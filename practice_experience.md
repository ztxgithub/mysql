# 影响数据库的性能

```shell
    1. 大量的并发和超高的CPU使用率
            大量的并发会使得数据库的连接数被占满(max_connections),超高的CPU使用率可能会宕机.
            
    2.网络IO
        避免无法连接到数据库
            (1) 减少从服务器的数量
            (2) 进行分级缓存
            (3) 避免使用"select *" 查询

```

# 数据库实战经验

```shell
    1. 最好不要在数据库主库(master)上进行数据库备份远程同步计划任务，大型活动前取消这类计划任务.因为这些同步计划任务会导致服务器的性能
        下降,使得其他正常的服务有堵塞现象.
        
    2.大多数数据库性能的问题可以通过对sql查询的优化来解决(慢查询问题)
```
